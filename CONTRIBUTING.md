# Contributing to VOO

Thank you for your interest in contributing to VOO!

## Ways to Contribute

### Report Bugs

If you find a bug in VOO, please open an issue on GitHub. Include:

- A clear description of the problem
- Steps to reproduce the issue
- Expected behavior vs. actual behavior
- Any relevant code snippets or error messages

### Propose Improvements

Have an idea to improve VOO? Open an issue to start a discussion for:

- Performance optimization ideas
- API design suggestions
- Documentation improvements
- New feature proposals

### Submit Code Changes

To contribute code changes:

1. **Fork the repository** on GitHub
2. **Create a feature branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes** and ensure they follow good programming practices.
4. **Run the test suite** to ensure your changes don't break existing functionality:
   ```bash
   tclsh test/test.tcl
   ```
5. **Commit your changes** with clear, descriptive commit messages
6. **Push to your fork** and open a **Pull Request** against the main repository
7. In your PR description, explain:
   - What problem does this solve or feature does it add?
   - Any relevant issue numbers (e.g., "Fixes #42")
   - Testing you've performed

## Development Guidelines

- Ensure all tests pass before submitting a PR
- Write clear commit messages
- Keep changes focused and logically separate
